# Glossary

This document defines project-specific vocabulary, abbreviations, and architectural labels to make the codebase legible.

## Core Domain Terms
- **Ankimon:** The name of the add-on, a portmanteau of Anki and Pokémon.
- **Review (Flashcard Review):** The core action in Anki where a user answers a flashcard. This action drives the progression of the Ankimon battle engine.
- **Encounter:** When a wild Pokémon appears for the user to battle.
- **Main Pokémon / User Pokémon:** The Pokémon currently active and fighting on behalf of the user. Persisted in `mainpokemon.json`.
- **Enemy Pokémon / Opponent:** The wild Pokémon currently being battled. Persisted transiently in memory.
- **PC / PC Box:** The UI (`pc_box.py`) where a user views and manages their collection of captured Pokémon.
- **Item Bag:** The UI and data structure (`itembag_path`) managing the user's consumable items (e.g., Potions, Pokeballs).

## Architectural Labels & Modules
- **poke_engine:** The complex sub-package located in `src/Ankimon/poke_engine/` responsible for simulating battle mechanics, calculating damage, and determining move effects. Based on "SirSkaro's Poke-Engine".
- **Singletons (`singletons.py`):** The file where all global state objects are instantiated so they can be injected into UI components.
- **Tracker (`AnkimonTracker`):** The class responsible for counting card reviews, tracking battle rounds, and triggering encounters.
- **Mutator:** A concept within the `poke_engine` that applies state instructions (damage, healing, status effects) to the battle state.
- **Instruction:** A tuple generated by the `poke_engine` describing a state change (e.g., `(MUTATOR_DAMAGE, OPPONENT, 15)`).

## Specific Variables & Acronyms
- **IVs (Individual Values):** Static, hidden stats that influence a Pokémon's maximum potential.
- **EVs (Effort Values):** Dynamic stats gained by defeating enemy Pokémon.
- **STAB (Same-Type Attack Bonus):** A damage multiplier applied when a Pokémon uses a move matching its own type.
- **`mypokemon.json`:** The JSON file acting as the primary persistence layer for the user's captured collection.
- **`aqt`:** The Anki Qt GUI module. Heavily imported across the UI classes to interface with Anki's windows.
